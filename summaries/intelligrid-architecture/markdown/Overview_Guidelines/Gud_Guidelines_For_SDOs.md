# Guidelines for Standards Developers

## Responding to IntelliGrid Architecture Recommendations

The IntelliGrid Project has identified a number of ‘[gaps’ in
existing standards and technologies](Rec_Recommendations_EPRI.htm). Members of standards bodies and other
interested parties should therefore use the IntelliGrid Architecture to
determine how best to close these gaps and develop linkages between
technologies. These proposed solutions should be actively encouraged and
discussed by the various users of the IntelliGrid Architecture to ensure a
consistent and complete resolution.

At the same time, the IntelliGrid Architecture must be
constantly updated to reflect changes in requirements and technologies.

IntelliGrid Architecture provides a loose, but specific, framework
illustrating the boundaries adjacent to individual standards efforts. By taking
note of these boundaries, standards organizations can tailor their fit at these
boundaries. This is possible without sacrificing or altering the requirements
of the specific standard, while allowing the results to be easily integrated
into a multi-application domain whole.
