# Quick Task 260619-g0p — Distribution Observability Source Appendix

## Objective

Add a new study-doc "appendix" to the AGMS patent material: an **exhaustive
inventory of distribution-level state-information sources** (devices, sensors,
systems, protocols) usable to reconstruct distribution-system state via virtual
sensing / DSSE, framed by the **ORACS Observability + Reachability** indexes
(Asset Portfolio Manager patent, WO 2024/211758, para [0209]).

Delivered as a **Markdown source + self-contained styled HTML** pair, matching the
existing patents-appendix pattern (`.planning/research/patents/AGMS-architecture.md`
+ `AGMS-architecture.html`), wired into `docs/build_site.py` so it copies into
`docs/architecture/` and is linked from the patents INDEX + study hub.

## Content (already drafted in conversation — reuse verbatim)

- **14-tier source inventory** (substation → feeder → phasor → AMI → DER →
  topology/model → pseudo-measurements → control-center systems → environmental →
  protection/event → asset-condition → comms substrate → protocols → non-utility),
  each row tagged "what it observes" + "how you reach it (protocol)".
- **Classification 1** — observability-tier prioritization (state-reconstruction
  leverage vs. availability/cost).
- **Classification 2** — mapping every source to DSSE measurement classes
  (real / PMU / AMI / virtual-zero-injection / pseudo + topology&params), with
  weight/covariance treatment.
- **Classification 3** — deployment placement (substation-edge / field FAD /
  ADMS-cloud) by ORACS reachability — the core of the deployment proposal.
- **Classification 4** — feeder archetypes (rural radial / suburban DER-heavy /
  urban networked) and which sources each realistically has.
