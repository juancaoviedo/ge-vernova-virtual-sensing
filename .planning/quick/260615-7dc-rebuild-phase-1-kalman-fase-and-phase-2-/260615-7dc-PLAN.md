## The central reframe (apply everywhere)

1. **The Kalman filter is a fusion vessel, not the point.** The hard, valuable problem is
   **manufacturing observability for under-measured distribution nodes** by *finding and fusing
   every source of side-information*. State estimation is reframed as an
   **information-sourcing-and-fusion** problem.
2. **Transmission vs. distribution.** Transmission gets redundancy **from space** (measurements
   `m` > states `n`; WLS over-determined; chi-squared / largest-normalized-residual bad-data
   detection works). Distribution is **under-determined** (`m ≪ n`; gain matrix `G = HᵀWH`
   **singular**; bad-data detection mostly **fails** because nearly every real measurement is
   *critical* — the **leverage/critical-measurement trap is the NORMAL condition**, not an edge case).
3. **The energy-transition double-edge.** DER penetration makes the physics **harder**
   (bidirectional, volatile, weather-coupled flow → larger process uncertainty) but **instruments
   the grid** (every smart inverter, battery, EVSE, AMI meter is a new telemetry source). Virtual
   sensing must *net this out positive*.
4. **The deliverable is a state estimate WITH a calibrated covariance**, and that covariance **IS
   the ORACS Observability index** that the director's AGMS architecture gates on (Asset Portfolio
   Manager, ref 1300). The estimator runs as an **Inspector scout** (`s_i`) on a Field Agent
   Device; because it is **recursive and local**, it keeps producing observability **in island mode
   when the WAN is down** — the AGMS resilience bet, satisfied by the filter's structure itself.
5. **Bridges to Juan's work (use honestly, throughout):** edge-ML load/PV forecasting (HEMS /
   Building Intelligence) = the **calibrated pseudo-measurement / temporal-prior generator**;
   **SI-MAPPER** ontology graph = the **topology / structural prior**; **OSED** edge runtime =
   the **Inspector-scout substrate**; **CVXPY MPC** = the **solve/simulate-before-commit consumer**
   that the estimate (with its covariance) feeds.

## Reference Appendix A — the FASE 3-bus feeder numeric trace (USE THESE EXACT NUMBERS in KAL-03)

**Feeder (radial):** bus 0 substation (V0 known, head flow metered, fast) — line 0–1 — bus 1
PV + smart inverter (self-reports P1, Q1; fast, every few s) — line 1–2 — bus 2 **dark** load
(AMI only: 15-min interval average, delayed; no real-time meter).

**Augmented-load state:** $x=[P_1,P_2]^\top$ (net real-power injection, kW; draw positive).
Reactive $Q_1,Q_2$ handled identically (suppressed in arithmetic; the inverter self-reports both
P and Q — that's why it's rich). Voltages recovered from injections via LinDistFlow.

**Channels:**
| Channel | model $h(x)$ | $R$ | rate |
|---|---|---|---|
| Head flow (SCADA) | $P_1+P_2+\text{loss}\approx P_1+P_2+5$ | $R_\text{head}=4$ (σ≈2) | fast |
| Inverter telemetry | $P_1$ | $R_\text{inv}=1$ (σ=1) | fast |
| AMI | $P_2$ (interval avg, delayed) | $R_\text{ami}=25$ (σ=5) | 15-min, stale |

(Head model folds line loss into a constant +5 kW to keep it linear; flag that in reality
$H_\text{head}\approx[1.05,1.08]$ and you'd relinearize.)

**Process model:** forecast ramp (diurnal profile, the `Bu` term) $v=[+0.3,+0.8]$ kW/min;
process noise $Q=\text{diag}(0.5,2.0)$ kW²/min. Predict: $\hat x\leftarrow\hat x+v\,\Delta t$,
$P\leftarrow P+Q\,\Delta t$.

**Ground truth (filter can't see):** $P_1^\text{true}=44$, $P_2^\text{true}=108$ (dark node heavier
than the forecast's 100; PV tapering faster than forecast so net draw higher).

**t₀ = 16:30 — prior (forecast only):**
$\hat x=[40,100]$, $P=\begin{bmatrix}25&0\\0&100\end{bmatrix}$ (σ1=±5, σ2=±10 kW).

**Event 1 — inverter update.** $z_\text{inv}=44$, $H=[1,0]$, $R=1$.
- $y=44-40=+4$; $S=25+1=26$; $K=[25/26,0]=[0.962,0]$.
- $\hat x=[40+0.962(4),\,100]=[\mathbf{43.85},100]$.
- $P\to\begin{bmatrix}0.96&0\\0&100\end{bmatrix}$ (σ1 ±0.98). P2 untouched — inverter sees only its own bus.

**Event 2 — head update (THE MONEY SHOT).** $z_\text{head}=157$ (=44+108+5), $H=[1,1]$, $R=4$.
- predicted head $=43.85+100+5=148.85$; $y=157-148.85=+8.15$.
- $PH^\top=[0.96,100]^\top$; $S=0.96+100+4=104.96$; $K=[0.0092,0.953]$.
- $\hat x=[43.85+0.0092(8.15),\,100+0.953(8.15)]=[43.92,\mathbf{107.77}]$.
- **$P_2$ jumped 100 → 107.8 toward the true 108 with NO sensor at bus 2** — head said "sum is high,"
  inverter had pinned P1, so the filter attributed the excess to the dark node (Kirchhoff coupling).
- $P\to\begin{bmatrix}0.95&-0.92\\-0.92&4.73\end{bmatrix}$. Two key facts: **$P_{22}$ collapsed
  100 → 4.73** (σ2 ±10 → **±2.17 kW**) — dark node observable via coupling, not a sensor; and the
  **off-diagonal went negative (−0.92)** — P1,P2 now negatively correlated because the head fixes
  their sum (the topology constraint, alive in the covariance).

**16:30→16:35 — comms gap (head SCADA drops 5 min), predict only:**
- $\hat x=[43.92+0.3(5),\,107.77+0.8(5)]=[45.4,111.8]$.
- $P=\begin{bmatrix}0.95+2.5&-0.92\\-0.92&4.73+10\end{bmatrix}=\begin{bmatrix}3.45&-0.92\\-0.92&14.73\end{bmatrix}$.
- σ2 ±2.17 → **±3.84 kW** — observability degraded the instant the sensor dropped, and the
  covariance reports it honestly. (Why the predict step / recursion matters; when head returns the
  next update snaps P2 back, same mechanics as Event 2.)

**16:45 — AMI arrives** (16:30–16:45 interval average ≈110, delayed/stale; true ~115):
- Applied to current state with $R=25$: $K=P_{22}/(P_{22}+25)\approx 3/28\approx0.11$ — a **weak pull**.
  Correctly so: stale + noisy. Its real jobs: (a) bound long-term drift, (b) feed **recalibration**
  ("this time-of-day runs hot" → update forecast $v$ and $Q$). Practitioner note: apply AMI to the
  **interval it describes** (delayed/lagged measurement / fixed-lag smoother), not bluntly to "now."

**Voltage map (the actual virtual-sensor output):** LinDistFlow, representative sensitivities
$V_2\approx 1.000 - 0.00012\,P_1 - 0.00035\,P_2$ (pu); propagate covariance
$\sigma_{V_2}=\sqrt{J P J^\top}$, $J=[-0.00012,-0.00035]$.
- Well-observed (post head update): $V_2 = 0.955 \pm 0.0008$ pu — comfortably above the 0.95 floor; AGMS idle.
- During comms gap (forecast-only, P inflated, load ramped): $V_2\approx 0.951 \pm 0.0013$ pu →
  lower 2σ ≈ 0.948, **under the limit**; $P(V_2<0.95)$ a few % → **trips the voltage-support CaCSM**.
  (The decision is driven by point estimate AND its growing uncertainty — and the uncertainty grew
  because observability degraded. Covariance = observability index = what AGMS gates on.)

## Reference Appendix B — the side-information taxonomy (USE in DSSE-02)

| Source | What it is, physically | Where it enters the estimator |
|---|---|---|
| Network topology (who connects to whom) | radial graph | defines `H` / power-flow coupling — lets the head measurement carry info downstream to unmetered nodes |
| Known injecting vs passive nodes | which buses have DER vs pure consumers | **zero-injection** nodes → near-zero-`R` virtual measurements (real info, no sensor); passive nodes → load sign known/bounded (one-sided prior); DER nodes → injection can reverse |
| Conductor/line params, ratings | line impedances | the *parameters* of the power-flow model (recovering them shrinks `Q`); also unlocks line-temp/asset-health. Distribution params are fuzzy → **LinDistFlow** + absorb error into `Q` |
| Power-flow equations | Kirchhoff coupling | the hard constraints `h(x)` tying measured & unmeasured nodes; distribution PF is *less* accurate (unbalanced, high R/X) |
| Load cyclicality / repeating profiles | diurnal/weekly seasonality | the **process model `F`** / known input `Bu` (FASE backbone): a year of 8 a.m.'s constrains tomorrow's 8 a.m. |
| Historical predictions + actuals + noise, indexed by time | the archive | **learned, time-varying `Q`,`R`**, conditioned on time-of-day/season/weather; calibrated pseudo-measurements |
| Smart meters (AMI) | slow (15-min), dense, delayed | anchors load pseudo-measurements with real energy data; low-rate measurement |
| EV chargers | possibly real-time, event-driven, large | high-rate measurement when present (OCPP/MQTT), absent between sessions |
| Batteries / smart inverters | self-reporting actuators (IEEE 1547 / 2030.5) | **sensor + actuator**: inverter knows its own terminal P,Q,V; DER adds measurement points for free |
| Different comm frequencies | asynchronous, multi-rate | native KF strength — **update on arrival with each source's own `R`; predict in between** |
| ML load/PV/DER forecasts | learned priors | high-quality **pseudo-measurements with calibrated covariance** = Juan's edge-ML work |
