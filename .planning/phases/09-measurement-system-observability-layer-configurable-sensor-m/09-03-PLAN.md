<objective>
Build the **data path** of the measurement runner `measure.py`: read ground truth (day `state`
or fault `fault_event`) via Plan-02 helpers, derive the per-class true values (including the
P_inj derivation that the `state` bucket does not store directly), select sensors per scenario
from `measure_config`, apply the energised gate (D-03), and write `meas` points per snapshot in
`snapshot` mode. Noise is applied as a thin pass-through placeholder here (gaussian-only, seeded);
the full three-model noise engine, multirate cadence, topology re-publish, and footprint report
are added in Plan 04 (which extends this same file).

Purpose: This establishes the deterministic per-snapshot loop and the correct true-value
derivation — the parts most likely to break (Pitfall 6: state bucket has no per-bus p_mw;
Pitfall 1: energised is a string tag; Pitfall 7: bus 17 dark in fault). Getting the data path
right first means Plan 04 only adds transforms.

Output: `system1-measurement-source/src/ieee33/measure.py` producing a populated `measurements`
bucket for `source=day` and `source=fault` in `snapshot` mode.
</objective>

P_inj DERIVATION (Pitfall 6 — state bucket bus has only vm_pu/va_degree):
- source=fault: P_inj = fault_bus.p_mw, Q_inj = fault_bus.q_mvar DIRECTLY (already net per-bus power).
  Sign convention: res_bus positive = net consumption. Emit as-is (document the convention in a comment).
- source=day: derive net injection per bus per step from profiles + network base loads:
    load_p[bus] = base_p_for_bus * load_pu(step)   (only for buses with a load; bus 0 has none)
    sgen_p[bus] = read_state_sgen value at that bus/step (DG buses 17,21,24,32)
    P_inj[bus]  = load_p[bus] - sgen_p[bus]   (net consumption convention, matching fault sign)
  Q_inj[bus] analogously using base_q and sgen q_mvar. Map net.load["bus"] -> base_p index so the
  right base load is applied to the right pandapower bus.
