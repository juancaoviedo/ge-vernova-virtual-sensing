<objective>
Fix UAT bug #3: AC-WLS raises `RankDeficientError` every step because the state vector
is sized 66 = 2×33 (it includes the electrical reference bus 0), and the assembled
measurement Jacobian only half-constrains the states.

This plan applies **D-11** consistently across `ac_model` / `estimators` / `estimate` /
`influx`: the estimation state is **64 = 2×32 free states for buses 1..32**, with **bus 0
as the fixed electrical reference** (|V|₀ regulated by the feeder OLTC, θ₀ = 0 reference).
The HV ext_grid bus 33 stays the pandapower slack embedded in Ybus as before. Re-audit the
snapshot → z/H assembly so the full `well_observed` real-only measurement set reaches H and
`G = HᵀWH` is full-rank (rank 64) WITHOUT pseudo padding (R5).

This is FOUNDATIONAL — plans 10-08 (injection_sensitivity + predict rewrite) and 10-10
(forecast consumer wiring) build on the 64/32 state definition established here.

Purpose: unblock WLS (R5/R10) and give EKF/UKF (R6/R7) a consistent 64-state contract.
Output: a full-rank 64-state estimation stack; WLS writes to the `estimates` bucket.
</objective>
