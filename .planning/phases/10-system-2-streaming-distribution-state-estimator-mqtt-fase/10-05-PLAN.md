<objective>
Build the separate scoring harness — the ONLY component allowed to read the `state`/`fault_event` oracle buckets. `score.py` joins the `estimates` bucket against the oracle and computes: per-bus |V| and angle RMSE; worst-case error at dark/pseudo buses; the flat/pseudo-only baseline comparison; the error-vs-observability contrast (`well_observed` vs `realistic_sparse`); covariance calibration (time-averaged NEES on the state vs the 95% χ² band, AND the per-step NIS in-band fraction computed from the faithful `nis_k`/`m_k` series that the recursive filters persisted in Plan 04); and the fault island-mode P-inflation check (σ_V higher in `faulted_isolated` than `pre_fault`, `restored` RMSE back within bar). It prints a falsifiable PASS/FAIL report and exits non-zero on any threshold miss. An additive `influx.read_estimates` (+ `read_estimate_system`) helper feeds it.

Purpose: This is the falsifiable acceptance gate for R10/R11/R12, and the structural enforcement of oracle separation (R9) — the estimator never imports this module. The NIS gate reads a true innovation sequence (`nis_k = y_kᵀ S_k⁻¹ y_k`, dimension `m_k`) persisted by the filters, not a reconstructed diagonal proxy.
Output: `uv run score` prints the report and exits non-zero on FAIL.
</objective>
