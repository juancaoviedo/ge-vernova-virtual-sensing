<objective>
Fix UAT bug #2: FASE predict crashes at `x_prev + S @ delta_p` because `S` was built from the
sparse MEASUREMENT Jacobian (columns = #injection measurements, e.g. 26) while `delta_p` is
per-bus (33). `S` and `delta_p` lived in different spaces.

This plan implements **D-10**: a correct FASE sensitivity `S = ∂x/∂p` over **bus injections**,
built from the power-flow injection Jacobian `J_p = ∂[P;Q]/∂[θ;|V|]` over non-slack buses,
`S = J_p⁻¹` reordered to the state layout. Its shape is `(64 × 2·n_nonslack)` — columns are bus
injections, INDEPENDENT of the measurement set. `FASEPredictor.predict` is rewritten to consume
`Δp = [ΔP;ΔQ]` and `Cov(ε)` supplied by the caller (the forecast stream, wired in 10-10), so it
no longer reads `prof_df` / a scalar `load_pu`.

Depends on 10-07: uses the 64-state (buses 1..32) definition established there.

Purpose: make EKF/UKF predict → update run across all snapshots without a dimension crash (R6/R7).
Output: dimension-consistent predict (`delta_p.shape[0] == S.shape[1]`).
</objective>

estimated; bus 0 = electrical reference (fixed); HV bus 33 = numerical slack (fixed in Ybus).
`jacobian_H(x, Ybus, meas_list)` returns `(m, 64)`.

Non-slack buses for the injection Jacobian = the SAME 32 estimated buses (1..32). So
`n_nonslack = 32` and `S.shape == (64, 64)` (2·32 injection columns). Δp = [ΔP over buses 1..32;
ΔQ over buses 1..32] length 64.

Power-flow injection Jacobian layout (standard polar Newton-Raphson):
  J_p = [[∂P/∂θ, ∂P/∂|V|], [∂Q/∂θ, ∂Q/∂|V|]]  over non-slack buses.
This is the SAME analytic machinery already in `jacobian_H` for P/Q injection rows — reuse
`_p_inj`/`_q_inj` partials rather than re-deriving. `S = J_p⁻¹`, then REORDER rows from the
[θ;|V|] / [P;Q] block layout into the interleaved state layout `[|V|₁,θ₁,...]` and the interleaved
injection layout `[P₁,Q₁,...]` (or document the chosen consistent ordering — the caller in 10-10
must build Δp in the MATCHING order).
