## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| any host on LAN → MQTT broker | An MQTT broker bound to 0.0.0.0 would accept unauthenticated publish/subscribe from any machine on the network |
| publish.py → broker | Local loopback producer; trusted in a local-dev interview-prep context |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Information disclosure / Elevation | Mosquitto broker network exposure | mitigate | Bind the broker to `127.0.0.1` ONLY — `ports: "127.0.0.1:1883:1883"` in docker-compose AND `listener 1883 127.0.0.1` in mosquitto.conf. Mirrors the existing InfluxDB/Grafana localhost-binding convention. Acceptance: `docker compose port mosquitto 1883` resolves to `127.0.0.1:1883`, never `0.0.0.0`. |
| T-10-02 | Spoofing | Anonymous broker auth | accept | `allow_anonymous true` — local-dev only, no PII, no external exposure (broker is localhost-bound per T-10-01). `security_enforcement` not configured for this project; documented as accepted. |
</threat_model>
