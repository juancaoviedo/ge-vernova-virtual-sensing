# Phase 7 — UI Design Contract

> Visual and interaction contract for the Integrated HTML Study Site. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This is a CONSISTENCY contract, not a greenfield design.** Every token below is extracted from the existing, authoritative hand-authored research HTML (`.planning/research/patents/AGMS-architecture.html` and `INDEX.html`). ## Spacing Scale

The existing CSS is built on a loose 4-/6-based rhythm. Declared values below are normalized to multiples of 4 for the new pages, mapped to the real values already in use. Where the existing CSS uses an off-grid value (e.g. 18px card padding, 36px grid gap), the new shared stylesheet MUST keep the existing value verbatim on copied/matched components so visual parity is exact — these are recorded as Exceptions.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline code padding, tight gaps |
| sm | 8px | TOC link padding (`5px 8px` → use 8px horizontal), compact spacing |
| md | 16px | Default paragraph/element spacing, TOC inner padding (`16px 14px`) |
| lg | 24px | List indent (`24px`), section inner padding rhythm |
| xl | 32px | Layout gaps, masthead horizontal padding (`30px 36px` region) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level bottom spacing (`padding 0 20px 80px` ≈ page floor) |

**Exceptions (verbatim from existing CSS — preserve exactly for parity, do NOT round):**
- Body line-height `1.7` (AGMS) / `1.65` (INDEX) — use `1.7` for converted notes to match the primary walkthrough.
- `.wrap` grid: `max-width:1180px`, `grid-template-columns:250px 1fr` (sidebar TOC + main), `gap:36px`, `padding:0 20px 80px`.
- `section.card` padding: `18px 30px 26px`; margin `18px 0`.
- `nav.toc` padding: `16px 14px`; TOC link padding `5px 8px`.
- `header.masthead` padding `30px 36px`; margin `28px 0 6px`.
- `.callout` padding `12px 18px`; `.juan` padding `11px 18px`.
- Responsive breakpoint: `max-width:900px` collapses the grid to one column and unsticks the TOC (`position:static`).
- Touch targets: not applicable — desktop/laptop study site, no mobile-first touch requirement beyond the 900px single-column reflow.

## Typography

Exactly 4 sizes / 2 weights for the new pages, mapped to the existing CSS. (Body weight is the system-font default 400; headings/strong use 600–700 via the browser default `<strong>`/`<h*>` — no extra weights introduced.)

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.7 |
| Label / meta / TOC | 13.5px | 400 | 1.5 |
| Heading (`h2.sec` / card titles) | 22px | 600 (semibold) | 1.3 |
| Display (`header.masthead h1`) | 28px | 600 (semibold) | 1.25 |

**Locked specifics from existing CSS (reuse exactly):**
- `h2.sec` 22px, bottom border `2px solid var(--accent-soft)`, color `#0c2a20`, `scroll-margin-top:16px`.
- `h3.sub` 17px, color `#13433a` (treat as a body-scale subheading, not a 5th declared size).
- `nav.toc h2` 11px uppercase, `letter-spacing:1.4px`, color `var(--muted)`.
- `code` inline 13px; `pre.diagram` 12.5px; table 14.5px, `thead th` 13px; `.juan` 14.5px; `footer` 12.5px.
- `strong` → `#0c2233`, `em` → `#33324a` (the existing emphasis colors — keep).
- pygments code blocks: 12.5–13px mono, line-height 1.5, to sit alongside `pre.diagram`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#eef1f5` (page bg) + `#fff` (`--panel`) | Page background and card/TOC/nav surfaces |

| Secondary accent | `--gold #b8860b`, `--callout #fff8e6` / `--callout-line #e0b84c` | Callout boxes only (not destructive — this is a study site, no destructive actions) |

**Accent (`--accent #0b6e4f`) reserved for:**
- Links (`a { color: var(--accent) }`)

NOT used for: body text, headings fill, large surfaces. Green stays a ~10% wayfinding/brand accent exactly as in the existing pages.

**Additional locked semantic tokens (carry over from INDEX.html for the hub):**
- `--start-bg #eaf3ff` / `--start-line #4a90d9` (blue) — INDEX uses this for "start here" emphasis; the hub MAY reuse it for a single "Start here" cue, otherwise leave unused. No other new colors.

## Copywriting Contract

This is a personal/interviewer-facing study site, so "CTA" = navigation affordances and section framing, not transactional buttons.

| Element | Copy |
|---------|------|
| Primary CTA (hub cards) | Verb-led card links: "Read the AGMS walkthrough", "Open the study notes", "See the demos" — each card title is the destination noun, link text is action-oriented |
| Hub eyebrow / masthead h1 | "GE Vernova Virtual Sensing — Interview Study Site" (masthead h1); `.meta` subtitle: "Architecture, study notes, and hands-on demos in one place." |
| Per-page "back to hub" | "← Back to study hub" (footer or top-of-page, every converted note + demo page) |
| Empty state heading | Not applicable — content is statically generated; there are no empty/dynamic data states. If a phase has zero notes (Phase 0/6, out of scope), simply omit the card — do not render an empty group. |
| Empty state body | n/a (no dynamic data) |
| Error state | Build-time only: the Python build script must fail loudly with "Note not found / could not convert: {path}" rather than emit a broken/blank HTML page. No runtime error UI (offline static site). |
| Broken-link guard | Acceptance requires zero 404s at `file://`; build script SHOULD validate that every hub link resolves to an emitted file. |
| Destructive confirmation | none — no destructive actions on a static site |

**Tone:** matches existing pages — plain-language, second-person where bridging ("Where your own experience maps in, you'll see a **▶ Juan** note"), study-companion voice. Keep `▶ Juan` bridge boxes (single full build, not redacted — per SPEC out-of-scope).

**Demo section copy structure (per D-12, sourced from each demo README):** each demo block leads with "What it is", "Why it was built (which interview gap it closes)", "What it demonstrates", then the inline result image + caption (D-10), then the embedded key-function code (D-11) with a "View full source →" link.

## Assumptions & Open Choices (autonomous-mode defaults)

These are the only genuinely open choices; all are flagged so the planner/executor can adjust:

1. **Body line-height 1.7** chosen over INDEX's 1.65 — matches the primary AGMS walkthrough (the dominant content style). Assumption.
2. **Single external `assets/site.css`** (vs inlined) — per CONTEXT discretion ("single external stylesheet preferred for maintainability"). The CSS is extracted verbatim from the existing `<style>` blocks plus the new pygments/hub-card/figure rules.
3. **Hub card hover** reuses the TOC link hover treatment (accent text + `--accent-soft` bg + accent left border) for visual consistency. Assumption.
4. **`--start-bg/--start-line` blue** carried over from INDEX but used at most once on the hub ("Start here" cue); default is to leave it unused to keep green as the sole accent. Assumption.
5. **MathJax tex-chtml** specific component/version left to planner (D-13 discretion); contract only requires offline `$…$`/`$$…$$` rendering.
6. **Pygments style** (e.g. a light style harmonized to `--code-bg #f4f6f8`) left to planner; contract requires it to visually coexist with `pre.diagram` and inline `code`.

No new typography, color, or layout primitives are introduced beyond the existing research-HTML design system plus the three additive components (highlighted code block, result figure, hub card grid) required by D-09/D-10/D-11.
