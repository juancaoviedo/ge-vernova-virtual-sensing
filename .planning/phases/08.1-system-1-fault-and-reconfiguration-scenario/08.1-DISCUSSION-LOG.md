## Validation & determinism

| Option | Description | Selected |
|--------|-------------|----------|
| Power-flow convergence per block | Every runpp converges; NaN check | ✓ |
| Radiality after restore | Radial post-restore; no closed loop | ✓ (after discussion) |
| Served-load drop + restored band | Load drops while isolated; restored vmin ≥0.95 (≥0.94 flagged) | ✓ |
| 40-point readback + determinism | Exactly 40 points; two runs identical | ✓ |

**User's choice:** All four gates enforced.
**Notes:** User questioned whether radiality is strictly required. Claude explained: not a solver
necessity (pandapower solves meshed nets), but (1) it matches the article's FLISR restoration,
(2) it's locked by SPEC req 4, and (3) it holds near-automatically here since the faulted line is
out of service — so the check is a cheap correctness guard. User chose **keep gate, drop the
deferred note** (no non-radial future-idea recorded).
