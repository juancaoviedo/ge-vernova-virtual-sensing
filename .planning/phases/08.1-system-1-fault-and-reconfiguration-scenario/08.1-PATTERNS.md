### `src/ieee33/fault_sim.py` (runner/driver, per-step CRUD)

**Analog:** `src/ieee33/sim.py`

**Imports pattern** (sim.py lines 1–30):
```python
"""
fault_sim.py
------------
<module docstring analogous to sim.py's>
"""

import sys
from datetime import datetime, timezone, timedelta

import pandapower as pp
from influxdb_client.client.write_api import SYNCHRONOUS

from ieee33 import config
from ieee33 import influx
from ieee33.network import build_enhanced_33bus
```
Note: `fault_sim.py` does NOT import `read_profiles` — the window is frozen at the
evening-peak operating point, not driven by a profile series.

**Network build + base-load capture pattern** (sim.py lines 131–133):
```python
net, trafo_idx = build_enhanced_33bus()
base_p = net.load["p_mw"].copy().values    # numpy array, MW at peak
base_q = net.load["q_mvar"].copy().values  # numpy array, MVAr at peak
```
`fault_sim.py` calls `build_enhanced_33bus()` ONCE, then mutates `in_service` flags per
block. Base loads are captured once from case33bw peak values; they are held constant
across all 40 steps (frozen evening-peak operating point, SPEC req 2 / D-09).

**InfluxDB connection + bucket setup pattern** (sim.py lines 137–141):
```python
client = influx.get_client()
influx.wait_for_influx()
influx.ensure_bucket(client, config.STATE_BUCKET)   # → replace STATE_BUCKET with FAULT_EVENT_BUCKET
```
`fault_sim.py` calls `ensure_bucket(client, config.FAULT_EVENT_BUCKET)` using the new
bucket name added to `config.py`.

**Forced-tap / run_control pattern** (sim.py lines 75–79 in `run_step()`):
```python
if forced_tap is not None:
    net.trafo.at[trafo_idx, "tap_pos"] = int(forced_tap)
    pp.runpp(net, algorithm="nr", calculate_voltage_angles=True, run_control=False)
else:
    pp.runpp(net, algorithm="nr", calculate_voltage_angles=True, run_control=True)
```
`fault_sim.py` REUSES this exact mechanism for D-12 (OLTC tap pinning during the
`faulted_isolated` block): the runner calls `pp.runpp` with `run_control=False` and
`net.trafo.at[trafo_idx, "tap_pos"] = <evening_peak_tap>` during isolated snapshots.
During `restored` it switches back to `run_control=True`.

**In-service line filter pattern** (sim.py lines 86–87 in `run_step()`):
```python
in_service_mask = net.line["in_service"]
res_line_inservice = net.res_line[in_service_mask].copy()
```
`fault_sim.py` applies the same filter AFTER mutating `in_service` flags per block.
This means the faulted line (idx 7) and dead-zone lines naturally drop from `res_line`
when their `in_service=False` flag is set.

**System-level scalar extraction pattern** (sim.py lines 89–106 in `run_step()`):
```python
total_load_mw  = float(net.res_load["p_mw"].sum())
total_gen_mw   = float(net.res_sgen["p_mw"].sum() + net.res_ext_grid["p_mw"].sum())
total_loss_mw  = float(net.res_line["pl_mw"].sum() + net.res_trafo["pl_mw"].sum())
vmin_pu        = float(net.res_bus["vm_pu"].min())
vmax_pu        = float(net.res_bus["vm_pu"].max())
slack_p_mw     = float(net.res_ext_grid["p_mw"].iloc[0])
slack_q_mvar   = float(net.res_ext_grid["q_mvar"].iloc[0])
```
Note: during `faulted_isolated`, `net.res_bus["vm_pu"].min()` will only see energised
buses (dead buses are absent from `res_bus` due to `in_service=False`). The runner
records this real convergent minimum, NOT 0.

**Per-step console table pattern** (sim.py lines 173–176 / 248–254):
```python
# Header (once before the loop):
print(
    f"\n{'Step':>4}  {'Phase':<18}  {'Vmin':>6}  {'Load MW':>8}  "
    f"{'Tap':>4}  {'Dead':>5}"
)
print("-" * 62)

# Per-step row (D-18):
print(
    f"{i:>4}  {phase:<18}  {vmin:>6.4f}  {served_load:>8.4f}  "
    f"{tap:>4}  {n_dead:>5}"
)
```
The fault runner prints: step index, phase label, vmin on energised buses, served load
(total_load_mw on in-service buses), tap position, dead-bus count. Mirror sim.py's
`f"{i:>4}  {str(ts)[:19]:<20}  {vmin:>6.4f} ..."` style.

**Per-step validation gate pattern** (sim.py lines 198–227):
```python
if not net.converged:
    issues.append(f"step {i:02d} ({ts}): power flow NOT converged")

if state["res_bus"]["vm_pu"].isna().any():
    issues.append(f"step {i:02d} ({ts}): NaN in bus vm_pu")
```
`fault_sim.py` adds a third per-step gate from D-15 (radiality in restored block):
```python
# D-15: after restore, verify no closed loop: tie closed AND faulted line open
if phase == "restored":
    tie_in_service = bool(net.line.at[config.FAULT_TIE_IDX, "in_service"])
    fault_in_service = bool(net.line.at[config.FAULT_LINE_IDX, "in_service"])
    if not (tie_in_service and not fault_in_service):
        issues.append(f"step {i:02d}: radiality check failed")
```

**Post-run read-back pattern** (sim.py lines 282–294):
```python
flux_readback = (

```
`fault_sim.py` mirrors this for D-17: read back bus_id="0" (always energised — slack
bus is never dead) from `fault_event`, assert exactly 40 points:
```python
flux_readback = (
    f'from(bucket: "{config.FAULT_EVENT_BUCKET}")\n'
    f'  |> range(start: {anchor_start}Z, stop: {anchor_stop}Z)\n'
    f'  |> filter(fn: (r) => r._measurement == "bus")\n'
    f'  |> filter(fn: (r) => r.bus_id == "0")\n'
    f'  |> filter(fn: (r) => r._field == "vm_pu")'
)
```

**write_state_step call pattern** (sim.py lines 235–246):
```python
influx.write_state_step(
    write_api,
    ts,
    state["res_bus"],
    state["res_line_inservice"],
```

```python
# ensure_bucket() — lines 78–97: idempotent; call BEFORE first write
def ensure_bucket(client: InfluxDBClient, name: str) -> None:
    buckets_api = client.buckets_api()
    if buckets_api.find_bucket_by_name(name) is None:
        org_id = client.organizations_api().find_organizations(org=config.INFLUXDB_ORG)[0].id
        buckets_api.create_bucket(bucket_name=name, org_id=org_id)
```

### write_state_step() — field/tag naming (forward contract)
**Source:** `src/ieee33/influx.py` lines 226–338 (full function)
**Apply to:** `fault_sim.py` new `write_fault_step()` helper
**Critical constraint:** Do NOT rename any existing fields. The fault writer MIRRORS the
existing schema byte-identically and EXTENDS it. Full existing schema:

```
measurement "bus"    tag bus_id   fields vm_pu, va_degree
measurement "line"   tag line_id  fields p_from_mw, q_from_mvar, p_to_mw, q_to_mvar,
                                         loading_percent, pl_mw, ql_mvar
measurement "sgen"   tag sgen_id  fields p_mw, q_mvar
measurement "system" (no tag)     fields total_load_mw, total_gen_mw, total_loss_mw,
                                         vmin_pu, vmax_pu, slack_p_mw, slack_q_mvar,
                                         tap_pos, shift_degree
```

New extensions in `write_fault_step()`:
```
measurement "bus"    add tag energised ("1" or "0")
                     dead-bus zero-fill: vm_pu=0.0, va_degree=0.0, energised="0"
measurement "line"   add tag energised ("1" or "0")
                     dead-line zero-fill: p_from_mw=0.0, q_from_mvar=0.0, p_to_mw=0.0,
                                          q_to_mvar=0.0, loading_percent=0.0,
                                          pl_mw=0.0, ql_mvar=0.0, energised="0"
measurement "sgen"   dead-zone sgen zero-fill: p_mw=0.0, q_mvar=0.0
measurement "event"  tag phase ∈ {pre_fault, faulted_isolated, restored}
                     fields faulted_line_id (int), tie_closed (int 0/1),
                            tie_id (int, -1 if open), n_dead_buses (int),
                            dead_buses (str, comma-joined bus ids e.g. "8,9,10,...")
```

```python
ts = row["_time"]
```

`fault_sim.py` adapts this for a fixed anchor (D-10): the anchor is the OPSD UTC

```python
anchor_ts = evening_peak_opsd_utc   # read from the 96-step state bucket at startup
```

This is fully deterministic (no wall-clock, no `datetime.now()`).
