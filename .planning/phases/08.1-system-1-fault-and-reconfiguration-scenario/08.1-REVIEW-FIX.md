## Fixed Issues

### CR-01: D-16 else-branch crash with ValueError when iso_loads is empty

**Files modified:** `system1-measurement-source/src/ieee33/fault_sim.py`
**Commit:** f783df8
**Applied fix:** Extracted `iso_min_str = f"{min(iso_loads):.4f}" if iso_loads else "N/A"` before the
print call so the `:.4f` format spec is only ever applied to a float, never to the string `'N/A'`.

---

### WR-01: Dead-zone sgen written twice per ISO step

**Files modified:** `system1-measurement-source/src/ieee33/influx.py`
**Commit:** 19140fe
**Applied fix:** Added `dead_sgen_set = set(dead_sgen_ids)` before the energised-sgen loop and a
`if idx in dead_sgen_set: continue` guard at the top of the loop body, so dead-zone sgen indices
are skipped in the energised loop and handled exclusively by the zero-fill loop below.

---

### WR-02: `sgen` measurement lacks `energised` tag — schema asymmetry breaks System 2 filtering

**Files modified:** `system1-measurement-source/src/ieee33/influx.py`
**Commit:** 19140fe (same commit as WR-01 — both touch the same sgen block)
**Applied fix:** Added `.tag("energised", "1")` to live sgen points in the energised loop and
`.tag("energised", "0")` to dead-sgen zero-fill points, mirroring the bus/line schema.

**ACTION REQUIRED before next run:** Adding the `energised` tag changes the InfluxDB series key
for the `sgen` measurement. The `fault_event` bucket must be deleted and `fault-sim` re-run to
avoid series-key conflicts. The orchestrator handles this (clean bucket before re-run).

---

### WR-03: `networkx` used directly but not declared as a direct dependency

**Files modified:** `system1-measurement-source/pyproject.toml`
**Commit:** d07947a
**Applied fix:** Added `"networkx>=3.0"` to `[project.dependencies]`, positioned after `pandapower==3.4.0`
with an inline comment in the commit message explaining the rationale. `uv run` rebuilt the package
cleanly and resolved `networkx` as a direct dependency.

---

### WR-04: `_select_restore_tie` swallows all exceptions without logging

**Files modified:** `system1-measurement-source/src/ieee33/fault_sim.py`
**Commit:** 7a32e0d
**Applied fix:** Changed `except Exception: continue` to
`except Exception as exc: print(f"  tie_idx={tie_idx}: exception during probe ({type(exc).__name__}: {exc})"); continue`
so any pandapower crash or unexpected error during probe is visible in the console before the loop
moves to the next candidate.
