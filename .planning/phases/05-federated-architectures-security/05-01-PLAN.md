3. **`## 2. Krum — Select the Update Closest to the Crowd`** — `**Mental model first.**` (honest updates cluster; a poisoned one is far from the cluster). Display-math Krum score, EXACT LaTeX required:
   `$$S(k) = \sum_{j \in \text{NN}_{n-f-2}(k)} \lVert \Delta w_k - \Delta w_j \rVert^2$$`
   where $\text{NN}_{n-f-2}(k)$ = the $(n-f-2)$ nearest neighbors of $k$ in gradient space. Selection: $k^* = \arg\min_k S(k)$. State the Multi-Krum variant (top $m$, then average). Assumption: knows/bounds $f$; requires $n \geq 2f+3$; complexity $O(n^2 d)$.
   Then a `### When Krum Fails` subsection (KAL-02 lines 110-129 pattern) with an `**Interview sentence:**` — vanilla Krum selects ONE update (Pitfall 3: do NOT say it averages; Multi-Krum averages a subset); needs a known/bounded $f$.

4. **`## 3. Coordinate-wise Median`** — `**Mental model first.**` (take the median per parameter dimension independently). Display-math, EXACT LaTeX required:
   `$$[\Delta w_\text{agg}]_i = \text{median}_k \left( [\Delta w_k]_i \right)$$`
   Why robust: median ignores extreme outliers as long as <half the clients are Byzantine; tolerates up to $n/2 - 1$ Byzantine (higher fraction than Krum). Then `### Limitation` (Pitfall 4 + Interview sentence): treats each dimension independently, ignoring gradient correlations — a sophisticated attacker can craft an update moderate in every dimension yet adversarial in the joint space. `**Interview sentence:** "Median for high Byzantine fraction; Krum for higher-dimensional joint robustness."`
