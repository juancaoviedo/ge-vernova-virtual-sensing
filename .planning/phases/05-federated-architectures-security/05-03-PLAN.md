**Helper functions** (each preceded by a `# ---` separator, one-line docstring), implementing 1-D mean estimation where `F_k(w) = mean((w - x)^2)`, `grad = 2*mean(w - x)`:
- `generate_clients(n_clients, true_means, n_samples, sigma)` → list of `(data: np.ndarray, n_k: int)`; client k samples `n_k` points from `N(true_means[k], sigma^2)`.
- `local_sgd(w, data, lr, epochs)` → run `epochs` plain gradient steps `w -= lr * 2*mean(w - data)`; return scalar local model. (FedAvg local training.)
- `fedprox_local_step(w_local, w_global, data, lr, epochs, mu)` → same loop but add the proximal gradient `mu*(w_local - w_global)` each step; return local model. (FedProx local training; the proximal term lives HERE, in the client objective.)
- `fedavg_aggregate(local_models, n_samples)` → weighted average `sum((n_k/n) * w_k)`.
- `krum_select(updates, f)` → for each update sum the squared distances to its `n - f - 2` nearest neighbors; return `np.argmin(scores)`. (Use the RESEARCH §"Code Examples" Krum snippet verbatim in spirit.)
- `coord_median(updates)` → `np.median(np.stack(updates), axis=0)` (works for scalar updates too).
- `inject_poison(updates, client_idx, magnitude)` → return a copy of `updates` with `updates[client_idx]` replaced by `(global - magnitude)` style poisoned value (a large negative shift outside the honest cluster).

**`run_demo()`** — single function, opens with `np.random.seed(42)`. Logic:
  1. Generate clients. Compute the true global optimum = weighted mean of TRUE_MEANS by N_SAMPLES (~1.22).
  2. **Plain FedAvg run:** N_ROUNDS rounds; each round every client runs `local_sgd` from the current global, server `fedavg_aggregate`. Record final estimate.
  3. **FedProx run:** identical but clients use `fedprox_local_step(..., mu=MU_FEDPROX)`. Record final estimate (should sit closer to the global optimum — damped drift).
  4. **Poisoned round (Byzantine demo):** take one round's set of client updates, `inject_poison` on POISON_CLIENT (shift −5.0). Compute three aggregations of that poisoned set: (a) plain `fedavg_aggregate` → corrupted; (b) `krum_select` → pick the honest update (poisoned client has the highest Krum score, rejected); (c) `coord_median` → ignores the outlier.
  5. Print the banner + contrast table.
