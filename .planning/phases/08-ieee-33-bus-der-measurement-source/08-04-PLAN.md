<objective>
Implement the deterministic 96-step quasi-static driver: read the day's profiles from InfluxDB once, sweep all 96 steps (scale loads by the load profile, set each DG by its solar/wind profile × nameplate, regulate via OLTC, run Newton-Raphson), capture the complete power-flow state per step, and persist every snapshot to the InfluxDB `state` bucket — with a per-step convergence/voltage-band validation gate.

Purpose: SPEC-4 (batch driver reading profiles from InfluxDB, no runtime network fetch), SPEC-5 (full ground-truth state per step), and SPEC-6 (96 snapshots persisted; queryable series). This is the heart of the "ground-truth measurement source" — the dataset System 2 will later consume.
Output: sim.py (the runner entry point).
</objective>
